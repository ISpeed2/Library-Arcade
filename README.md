# Техническое задание на проект "Демонстрационный стенд библиотеки Arcade"

## 1. Общая информация

**Название проекта:** Демонстрационный стенд библиотеки Arcade  
**Цель проекта:** Создать модульное консольное приложение на Python для наглядной демонстрации ключевых возможностей библиотеки `arcade`: рисования примитивов, анимации, обработки событий и работы со спрайтами.

## 2. Описание задачи (что нужно сделать)

Необходимо разработать модульное приложение, которое через консольное меню запускает интерактивные графические демонстрации возможностей Arcade. Каждая демонстрация открывает отдельное окно и наглядно показывает конкретный аспект библиотеки. Приложение должно работать без дополнительных зависимостей кроме самой библиотеки `arcade`.

## 3. Функциональные требования (что система должна уметь)

1. Система должна предоставлять текстовое меню для выбора демонстрации.
2. Система должна демонстрировать рисование всех базовых примитивов Arcade: линий, прямоугольников, окружностей, треугольников, многоугольников, дуг, звёзд.
3. Система должна демонстрировать анимацию объекта с отражением от границ экрана и эффектом шлейфа.
4. Система должна показывать обработку событий клавиатуры и мыши в реальном времени.
5. Система должна демонстрировать работу со спрайтами через `arcade.SpriteList`.
6. Система должна предоставлять полную последовательную демонстрацию всех возможностей.
7. Система должна корректно обрабатывать неверный ввод пользователя в меню.
8. Выход из каждой демонстрации должен осуществляться по клавише ESC или закрытию окна.

## 4. Нефункциональные требования (ограничения)

1. Язык программирования: Python 3.8+
2. Используемые библиотеки: только `arcade`
3. Интерфейс выбора демонстрации: консольный (CLI)
4. Архитектура: модульная (разделение на core, utils, logic, app)
5. Код должен соответствовать PEP 8 и содержать docstrings
6. Проект должен запускаться на Windows, Linux и macOS

## 5. Требования к интерфейсу

1. При запуске программы отображается главное консольное меню с нумерованными пунктами.
2. Меню содержит следующие пункты:
   * Геометрические примитивы
   * Анимация — прыгающий шар
   * Обработка событий (клавиши + мышь)
   * Спрайты и SpriteList
   * Полная демонстрация всех возможностей
   * Выход
3. Пользователь выбирает пункт, вводя цифру от 0 до 5.
4. Каждая демонстрация открывает графическое окно 800×600 пикселей.
5. После закрытия окна программа возвращается в главное меню.

## 6. Входные и выходные данные

**Входные данные:**

* Цифра от 0 до 5 для выбора демонстрации
* В демо событий: движение мыши, нажатия клавиш R/G/B/ПРОБЕЛ/↑/↓, клик мыши
* В демо спрайтов: нажатие ПРОБЕЛ для добавления нового шара

**Выходные данные:**

* Графические окна Arcade с интерактивными демонстрациями
* Консольные сообщения с описанием текущей демонстрации и управления
* Счётчик событий и текущие параметры объектов (скорость, форма, размер)

## 7. Критерии приёмки (когда работа считается выполненной)

Работа считается выполненной, если:

1. Реализованы все функциональные требования из пункта 3.
2. Программа запускается командой `python app.py` без ошибок.
3. Все четыре демонстрации открывают окно и отображают корректную графику.
4. Анимация работает плавно с использованием `delta_time`.
5. Демонстрация событий корректно реагирует на мышь и клавиатуру.
6. Спрайты отражаются от границ экрана и добавляются по ПРОБЕЛ.
7. Код разделён на модули согласно архитектуре (core, utils, logic, app).
8. Проект загружен в Git-репозиторий с полной структурой файлов.
9. В репозитории присутствует файл README.md с описанием проекта.

├── utils.py         # Вспомогательные функции: цвета, фигуры, таймер
├── logic.py         # Логика демонстраций (классы окон)
├── app.py           # Главное приложение с консольным меню
├── requirements.txt # Зависимости проекта
└── README.md        # Документация проекта

## Описание модулей

### 1. `core.py` — Ядро системы

Содержит базовый класс и вспомогательную функцию запуска:

* **`BaseWindow`** — базовый класс, наследующийся от `arcade.Window`. Реализует закрытие по ESC и общий интерфейс для всех демонстрационных окон
* **`run_demo(window)`** — запускает игровой цикл переданного окна

### 2. `utils.py` — Утилиты

Содержит вспомогательные функции:

* **`random_color()`** — генерирует случайный насыщенный цвет
* **`color_from_angle(angle_deg)`** — плавный HSV-переход цвета по углу (для анимации)
* **`draw_star(cx, cy, outer, inner, points, color)`** — рисует произвольную звезду
* **`print_demo_header(name)`** — форматированный вывод заголовка демонстрации
* **`Timer`** — класс для замера времени

### 3. `logic.py` — Логика демонстраций

Содержит классы-окна и функции-запускатели:

* **`PrimitivesWindow`** — все примитивы Arcade на одном экране
* **`AnimationWindow`** — прыгающий шар с шлейфом и сменой цвета
* **`EventsWindow`** — интерактивная фигура, управляемая мышью и клавиатурой
* **`SpritesWindow`** — 15 спрайтов-шаров с физикой отражения
* **`demo_*()`** — функции-запускатели для каждого окна

### 4. `app.py` — Главное приложение

Содержит:

* **`show_menu()`** — функция отображения главного меню
* **`main()`** — главный цикл программы с обработкой выбора пользователя

## Демонстрации

### 1. Геометрические примитивы

Отображает полный набор 2D-примитивов Arcade на одном экране: цветные линии-веер, прямоугольник с обводкой, окружность, эллипс, треугольник, правильный шестиугольник, пятиконечная звезда и дуга. Каждый примитив подписан.

### 2. Анимация — прыгающий шар

Шар движется по экрану, отражаясь от границ. Скорость задаётся через `delta_time` для плавной анимации независимо от FPS. Цвет шара плавно меняется через HSV-переход. За шаром остаётся полупрозрачный шлейф из 25 позиций. В углу отображается текущая скорость.

### 3. Обработка событий

Интерактивная демонстрация: фигура следует за курсором мыши, клавиши R/G/B меняют цвет, ПРОБЕЛ переключает форму между кругом и квадратом, стрелки ↑/↓ изменяют размер, ЛКМ назначает случайный цвет. Счётчик событий и текущие параметры отображаются на экране.

### 4. Спрайты и SpriteList

Создаёт 15 спрайтов-шаров (`arcade.SpriteCircle`) разного размера и цвета. Каждый спрайт имеет случайную начальную скорость и отражается от границ экрана. По нажатию ПРОБЕЛ добавляется новый шар. Демонстрирует эффективную групповую отрисовку через `SpriteList.draw()`.

### 5. Полная демонстрация

Запускает все четыре демонстрации последовательно.

## Ключевые концепции

### 1. Наследование от arcade.Window

* **Проблема:** нужно создать окно с собственной логикой рендеринга и событий
* **Решение:** наследование от `arcade.Window` и переопределение `on_draw()`, `on_update()`, `on_key_press()`
* **Пример:** `class AnimationWindow(BaseWindow)` — добавляет физику шара в `on_update`

### 2. delta_time для плавной анимации

* **Проблема:** скорость анимации зависит от FPS компьютера
* **Решение:** `on_update(delta_time)` передаёт время прошедшего кадра в секундах
* **Пример:** `self.x += self.vx * delta_time` — движение не зависит от FPS

### 3. SpriteList для групповой отрисовки

* **Проблема:** отрисовка 100 спрайтов по одному медленна
* **Решение:** `SpriteList` батчит отрисовку на GPU за один вызов
* **Пример:** `self.balls.draw()` рисует все 15 спрайтов одним вызовом

### 4. Система событий Arcade

* **Проблема:** нужно реагировать на разные виды ввода
* **Решение:** переопределение методов `on_key_press()`, `on_mouse_motion()`, `on_mouse_press()`
* **Пример:** `on_mouse_motion(x, y, dx, dy)` вызывается при каждом движении мыши

### 5. BaseWindow и повторное использование кода

* **Проблема:** каждое демо нуждается в закрытии по ESC и общей инициализации
* **Решение:** базовый класс `BaseWindow` с общей логикой, от которого наследуются все демо
* **Пример:** добавить новое демо = создать класс, унаследовав от `BaseWindow`

## Заключение

Данный проект демонстрирует практическую пользу библиотеки `arcade` для создания 2D-графики и интерактивных приложений на Python. Arcade предоставляет более высокоуровневый и Pythonic API по сравнению с pygame, что делает её идеальной для обучения игровому программированию. Модульная архитектура проекта позволяет легко добавлять новые демонстрации, не изменяя существующий код.
